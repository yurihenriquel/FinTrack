# FinTrack
Sistema Web de Controle de Gastos

---

## Etapa 1 — Estrutura Inicial e Página Index

### Objetivo da etapa
Criar a estrutura base do projeto e desenvolver a página inicial (`index.html`), aplicando conceitos fundamentais de HTML5 e CSS.

### Estrutura trabalhada
- Separação entre frontend e backend
- Organização de pastas por responsabilidade
- Criação do arquivo `index.html`
- Criação do arquivo `style.css`

### Conceitos aplicados

#### HTML
- Estrutura básica do documento (`<!DOCTYPE html>`)
- Uso de `header`, `main`, `section` e `footer`
- Links de navegação entre páginas
- Boas práticas de semântica HTML5

#### CSS
- Reset de estilos
- Estilização global
- Flexbox para centralização
- Criação de botões com efeito hover
- Layout em formato de card

### Resultado
Uma página inicial funcional, estilizada e organizada, servindo como ponto de entrada do sistema FinTrack.

---

## Etapa 2 — Tela de Cadastro de Gastos

### Objetivo da etapa
Criar um formulário HTML para cadastro de gastos financeiros, aplicando validações nativas do HTML5.

### Conceitos aplicados
- Formulários HTML
- Inputs do tipo number, date e text
- Validação com required
- Integração futura com JavaScript
- Reutilização de estilos CSS

### Resultado
Tela funcional para cadastro de gastos, preparada para comunicação com o backend via JavaScript.

---

## Etapa 3 — JavaScript e Captura do Formulário

### Objetivo da etapa
Implementar a lógica em JavaScript para capturar os dados do formulário de cadastro de gastos.

---

### Conceitos aplicados
- DOM (Document Object Model)
